# Data

Raw data files are not included in this repository (total size > 500 MB). Please download them directly from Kaggle.

## Download Instructions

1. Accept the competition rules at:  
   https://www.kaggle.com/competitions/m5-forecasting-accuracy

2. Install the Kaggle CLI:
   ```bash
   pip install kaggle
   ```

3. Download and unzip into `data/raw/`:
   ```bash
   kaggle competitions download -c m5-forecasting-accuracy
   unzip m5-forecasting-accuracy.zip -d data/raw/
   ```

## Required Files

Place the following files under `data/raw/`:

```
data/
└── raw/
    ├── sales_train_evaluation.csv   # ~120 MB  Daily unit sales (wide format, 1,941 days)
    ├── sales_train_validation.csv   # ~120 MB  Sales through the validation cutoff
    ├── sell_prices.csv              # ~200 MB  Weekly item sell price per store
    ├── calendar.csv                 #  ~100 KB  Date metadata, SNAP flags, event names
    └── sample_submission.csv        #    ~5 MB  Submission format reference
```

## Weather Data (Exploratory Only)

`02_external_data_and_features.ipynb` loads a `weather.csv` file with daily temperature records for the three M5 states (CA, TX, WI), assembled from public NOAA data. This file is not included in the repository. The weather features produced no meaningful RMSE improvement (< 0.0001 vs. baseline) and are **not used in the final model**. The weather join code is retained in notebook 02 for reproducibility.

If you want to reproduce the weather experiment, the expected format is:

| Column | Description |
|---|---|
| `STATION` | NOAA station ID (mapped to CA / TX / WI) |
| `DATE` | Date string (YYYY-MM-DD) |
| `PRCP` | Precipitation (inches) |
| `SNOW` | Snowfall (inches) |
| `TMIN` | Min temperature (°F or °C — match your source) |
| `TMAX` | Max temperature (°F or °C) |

Place the file at `data/raw/weather.csv` and update the path in the notebook's configuration cell.
